<task type="auto">
  <name>Task 2: Create TypeScript types, typed event bus, logger, and config validation</name>
  <files>src/types/events.ts, src/types/api.ts, src/types/metrics.ts, src/core/event-bus.ts, src/core/logger.ts, src/core/config.ts</files>
  <action>
    1. Create `src/types/api.ts`:
       - Define `ErrorType` as union: `'rate_limited' | 'server_error' | 'client_error' | 'timeout'`
       - Define `ClassifiedError` interface: `{ type: ErrorType; statusCode?: number; retryAfterMs?: number; message: string }`
       - Define `CortexResponse` interface: `{ content: string; model: string; promptTokens: number; completionTokens: number; latencyMs: number }`
       - Export all types

    2. Create `src/types/events.ts`:
       - Import types from api.ts
       - Define `ApiRequestEvent`: `{ model: string; messageCount: number; timestamp: number }`
       - Define `ApiResponseEvent`: `{ latencyMs: number; promptTokens: number; completionTokens: number; model: string; content: string }`
       - Define `ApiErrorEvent`: `{ type: ErrorType; statusCode?: number; retryAfterMs?: number; message: string; latencyMs: number }`
       - Define `EventMap` interface mapping event names to argument tuples:
         - `'api:request'`: `[ApiRequestEvent]`
         - `'api:response'`: `[ApiResponseEvent]`
         - `'api:error'`: `[ApiErrorEvent]`
       - Export all types

    3. Create `src/types/metrics.ts`:
       - Define `LatencyMetric`: `{ latencyMs: number; timestamp: number }`
       - Define `TokenUsage`: `{ promptTokens: number; completionTokens: number }`
       - Export all types (these will be expanded in later phases)

    4. Create `src/core/event-bus.ts`:
       - Import `EventEmitter` from `'events'`
       - Import `EventMap` from types
       - Create a `TypedEventEmitter<T>` class wrapping `EventEmitter`:
         - `emit<K extends keyof T & string>(event: K, ...args: T[K]): boolean`
         - `on<K extends keyof T & string>(event: K, listener: (...args: T[K]) => void): this`
         - `once<K extends keyof T & string>(event: K, listener: (...args: T[K]) => void): this`
         - `off<K extends keyof T & string>(event: K, listener: (...args: T[K]) => void): this`
       - Set `maxListeners` to 50 on the internal emitter (prevents warning in later phases with multiple subscribers)
       - Export singleton: `export const eventBus = new TypedEventEmitter<EventMap>()`

    5. Create `src/core/logger.ts`:
       - Import `pino` from `'pino'`
       - Detect environment: `const isDev = process.env.NODE_ENV !== 'production'`
       - Create logger with:
         - Level from `process.env.LOG_LEVEL ?? 'info'`
         - In dev: use `pino-pretty` transport with `colorize: true` and `translateTime: 'HH:MM:ss'`
         - In production: no transport (raw JSON)
       - Export: `export const logger = pino({ ... })`

    6. Create `src/core/config.ts`:
       - Import `dotenv/config` (side-effect import to load .env)
       - Define `AppConfig` interface: `{ apiKey: string; baseUrl: string; model: string }`
       - Create `validateConfig()` function:
         - Check `process.env.CORTEX_API_KEY`
         - If missing: print clear error message to stderr (NOT throw):
           ```
           Error: CORTEX_API_KEY environment variable is not set.
           Set it in your .env file or export it in your shell:
             export CORTEX_API_KEY=your-key-here
           ```
         - Call `process.exit(1)` — no stack trace
         - If present: return `{ apiKey, baseUrl: 'https://cortex.nfinitmonkeys.com/v1', model: 'default' }`
       - Export `validateConfig` and `AppConfig`

    7. Run `npx tsc --noEmit` to verify all files compile with strict mode.
  </action>
