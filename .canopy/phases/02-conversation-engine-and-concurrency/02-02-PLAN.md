**GREEN phase -- create `src/conversation/manager.ts`:**

```typescript
import { createConversationRunner } from './runner.js';
import { eventBus } from '../core/event-bus.js';
import { logger } from '../core/logger.js';
import { setTimeout } from 'timers/promises';
import type { ManagerConfig, TestRunResult, ConversationResult } from '../types/conversation.js';

export function createConversationManager(config: ManagerConfig) {
  const {
    numConversations,
    turnsPerConversation,
    makeRequest,
    rampUpDelayMs = 200,
    drainTimeoutMs = 30_000,
  } = config;

  let stopping = false;
  let running = false;

  async function start(): Promise<TestRunResult> { ... }
  async function stop(): Promise<void> { ... }

  return { start, stop };
}
```

Implementation logic for `start()`:

1. Set `running = true`, emit test:lifecycle 'starting'
2. Emit test:lifecycle 'running'
3. Create an array of promises. For each conversation (1 to N):
   a. Create a ConversationRunner with:
      - conversationId: i (1-indexed sequential)
      - turnsPerConversation: from config
      - makeRequest: from config
      - shouldStop: () => stopping
   b. Launch runner.run() and push the promise (do NOT await individual runners)
   c. Log the conversation launch with timestamp and ID
   d. If not last conversation AND rampUpDelayMs > 0: await a delay of `rampUpDelayMs + Math.random() * 0.5 * rampUpDelayMs`
4. Wait for all promises: `const settled = await Promise.allSettled(promises)`
5. Collect results from settled (fulfilled values are ConversationResult, rejected should not happen since runners catch internally)
6. Build and return TestRunResult

Implementation logic for `stop()`:

1. Set `stopping = true`
2. Emit test:lifecycle 'stopping' with conversationsActive count
3. Emit test:lifecycle 'draining'
4. The active runners will check `shouldStop()` before each new turn and exit
5. If there's a stored reference to the main Promise.allSettled, race it against a drain timeout:
   ```typescript
   await Promise.race([
     activeRunPromise,
     setTimeout(drainTimeoutMs).then(() => {
       logger.warn({ event: 'drain_timeout', drainTimeoutMs }, 'Drain timeout reached');
     }),
   ]);
   ```
6. Emit test:lifecycle 'stopped'

**Key design for stop/drain coordination:**
The `start()` method stores its allSettled promise so `stop()` can race against it. Since `stopping = true` causes runners to skip new turns, most runners will complete quickly. The drain timeout catches cases where an in-flight API request is very slow.
