<!-- Key contracts from Plan 01 (server) and Plan 02 (client foundation) -->

Server REST API (from Plan 01):
```
POST /api/test/start
  Body: { numConversations: number, turnsPerConversation: number, concurrency: number }
  Response: 202 { status: 'started' }
  Error: 409 { error: 'Test already running' }

POST /api/test/stop
  Response: 200 { status: 'stopping' }

GET /api/test/status
  Response: 200 { status: 'idle' | 'running' | 'stopping' }
```

Zustand Store (from Plan 02 — client/src/store/useTestStore.ts):
```typescript
// State
config: TestConfig;                           // { numConversations, turnsPerConversation, concurrency }
testStatus: TestStatus;                       // 'idle' | 'running' | 'stopping' | 'stopped'
conversations: Map<number, ConvRow>;          // Per-conversation live state
latencyHistory: LatencyPoint[];               // { time, p50, p95, p99 } array for chart
tokenHistory: TokenPoint[];                   // { time, promptTokens, completionTokens } array for chart
errorCounts: Record<string, number>;          // Error type -> count
summary: unknown | null;                      // TestSummary after completion
throughput: { requestsPerSecond, tokensPerSecond } | null;

// Actions
setConfig(cfg: Partial<TestConfig>): void;
setTestStatus(s: TestStatus): void;
upsertConversation(partial): void;
addLatencyPoint(pt): void;
addTokenPoint(pt): void;
incrementError(type: string): void;
setThroughput(t): void;
setSummary(s): void;
reset(): void;

// Types
ConvRow: { conversationId, status: ConvStatus, currentTurn, turnsTotal, lastLatencyMs, totalTokens, errors }
ConvStatus: 'idle' | 'active' | 'completed' | 'errored'
LatencyPoint: { time: number, p50: number, p95: number, p99: number }
TokenPoint: { time: number, promptTokens: number, completionTokens: number }
```

- Monospace font for metric values, system font for labels
- Minimal chrome — data density over decoration
- Latency chart is the hero element (largest chart panel)

**IMPORTANT:** The throughput values need to be updated live during the test, not just at the end. The SSE hook currently gets `conversation:turn:complete` events. To compute live throughput, add a computation in useSSE.ts: track total requests and total elapsed time since test start, then compute req/s and tok/s. Call `setThroughput()` on each batch. If this requires changes to the store or hook from Plan 02, make them here.

Actually, the simplest approach: have the SSE hook track `testStartTime` (captured from `test:lifecycle` `running` event), and on each `conversation:turn:complete`, compute:
- elapsed = Date.now() - testStartTime
- requestsPerSecond = totalRequestsSoFar / (elapsed / 1000)
- tokensPerSecond = totalTokensSoFar / (elapsed / 1000)

Store these module-level in useSSE.ts and call `setThroughput()`. Update useSSE.ts if needed.
