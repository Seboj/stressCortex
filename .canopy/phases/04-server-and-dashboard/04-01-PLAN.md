**Step 2: Create `src/server/test-controller.ts`.**

This class wraps the existing `createConversationManager` and `MetricsCollector` so the server can start/stop tests independently, multiple times, without restarting the process.

Implementation:

```typescript
export type ControllerStatus = 'idle' | 'running' | 'stopping';

export class TestController {
  private status: ControllerStatus = 'idle';
  private manager: ReturnType<typeof createConversationManager> | null = null;
  private collector: MetricsCollector | null = null;
}
```

Key methods:

- `getStatus(): ControllerStatus` — returns current status.

- `async start(params: { numConversations: number; turnsPerConversation: number; concurrency: number }): Promise<void>` — Must throw Error if `this.status !== 'idle'`. Sets status to `'running'`. Creates fresh `MetricsCollector` and `createConversationManager`. Calls `validateConfig()` and `createCortexClient(config)` to get `makeRequest`. The `concurrency` param maps to `numConversations` in ManagerConfig (concurrency = how many run simultaneously, which is the same as numConversations since all launch with stagger). Calls `manager.start()` in a fire-and-forget `void` promise chain — do NOT await it, because the REST endpoint returns 202 immediately. In the `.then()` callback: call `collector.getSummary()`, emit `metrics:summary` event via eventBus, call `collector.destroy()`, null out manager and collector, set status back to `'idle'`.

- `async stop(): Promise<void>` — If not running, return silently. Set status to `'stopping'`. Call `await this.manager.stop()`.

**IMPORTANT:** The `makeRequest` type cast from OpenAI types to `{role, content}` must be applied here, exactly as done in the current `src/index.ts` (line 89-92). Copy that same cast pattern.

Export a singleton: `export const testController = new TestController();`

**Step 1: Create `src/server/sse-bridge.ts`.**

The SseBridge class subscribes to the event bus and batches updates to connected SSE clients every 200ms (SERV-03).

```typescript
import { eventBus } from '../core/event-bus.js';
import type { ServerResponse, IncomingMessage } from 'http';
```

Class `SseBridge`:
- `private clients = new Set<ServerResponse<IncomingMessage>>()`
- `private queue: Array<{ type: string; [key: string]: unknown }> = []`
- `private flushInterval: ReturnType<typeof setInterval> | null = null`

Method `start()`:
- Subscribe to ALL event types that the dashboard needs. For each, push an object with `{ type: 'eventName', ...eventData }` to the queue:
  - `conversation:start` → `{ type: 'conversation:start', ...evt }`
  - `conversation:turn:complete` → `{ type: 'conversation:turn:complete', ...evt }`
  - `conversation:complete` → `{ type: 'conversation:complete', ...evt }`
  - `test:lifecycle` → `{ type: 'test:lifecycle', ...evt }`
  - `metrics:summary` → `{ type: 'metrics:summary', ...evt }`
  - `api:error` → `{ type: 'api:error', ...evt }`
- Start a `setInterval` at 200ms calling `this.flush()`.

Method `addClient(res: ServerResponse<IncomingMessage>)`:
- Add to clients set.
- On `res.on('close', () => this.clients.delete(res))`.

Method `private flush()`:
- If queue is empty or no clients, return.
- Drain queue atomically: `const batch = this.queue.splice(0)`.
- Build SSE payload: `const payload = 'data: ' + JSON.stringify(batch) + '\n\n'`.
- Write to each client in a try/catch; delete client on error.

Method `stop()`:
- Clear the interval.

Method `destroy()`:
- Stop the interval.
- Remove all eventBus listeners (store bound handlers in constructor like MetricsCollector does — use `eventBus.off()` for each).

Export singleton: `export const sseBridge = new SseBridge()`.

**Step 2: Create `src/server/routes/events.ts`.**

```typescript
import type { FastifyInstance } from 'fastify';
import { sseBridge } from '../sse-bridge.js';

export async function eventsRoute(fastify: FastifyInstance): Promise<void> {
  fastify.get('/api/events', async (request, reply) => {
    reply.raw.writeHead(200, {
      'Content-Type': 'text/event-stream',
      'Cache-Control': 'no-cache',
      'Connection': 'keep-alive',
      'Access-Control-Allow-Origin': '*',
    });
    sseBridge.addClient(reply.raw);
    // Block the handler — connection stays open until client disconnects
    await new Promise<void>((resolve) => {
      reply.raw.on('close', resolve);
    });
  });
}
```
