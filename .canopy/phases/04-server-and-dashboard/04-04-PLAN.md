  <action>
**Step 1: Build the React client for production.**

```bash
cd client && npm run build
```

This should produce `client/dist/` with `index.html` and bundled JS/CSS.

Verify the build output exists:
```bash
ls -la client/dist/
```

**Step 2: Verify the Fastify server serves the built client.**

Start the server:
```bash
CORTEX_API_KEY=test-key tsx src/index.ts &
```

Wait for it to listen, then:
```bash
curl -s http://localhost:3001/ | head -20
```

Should return the HTML content of the dashboard (from `client/dist/index.html`).

Also verify API endpoints:
```bash
curl -s http://localhost:3001/api/test/status
```

Should return `{"status":"idle"}`.

And verify SSE endpoint opens:
```bash
curl -s -N --max-time 2 http://localhost:3001/api/events 2>/dev/null || true
```

Should connect without error (will timeout after 2s since no events are flowing).

Kill the background server after verification.

**Step 3: Fix any issues found during integration.**

Common issues to check:
- `@fastify/static` path to `client/dist` is correct relative to compiled server location
- SPA fallback (setNotFoundHandler) serves index.html for non-API routes
- CORS headers don't interfere when serving from same port
- Client bundle correctly references `/api/*` paths (no hardcoded localhost:5173)

If the `@fastify/static` root path is wrong (because the server runs via `tsx` from `src/` not from `dist/`), adjust the path in `src/server/index.ts` to use a path relative to the project root. For example:
```typescript
// When running via tsx, import.meta.dirname is src/server/
// client/dist is at ../../client/dist relative to src/server/
const clientDistPath = path.join(import.meta.dirname, '../../client/dist');
```

Or use the project root:
```typescript
const projectRoot = path.join(import.meta.dirname, '../..');
const clientDistPath = path.join(projectRoot, 'client/dist');
```

Verify the path resolves correctly.

**Step 4: Add a convenience script to package.json (root).**

Add scripts to the root `package.json`:
```json
{
  "scripts": {
    "start": "tsx src/index.ts",
    "dev": "tsx watch src/index.ts",
    "dev:client": "cd client && npm run dev",
    "build:client": "cd client && npm run build",
    "build": "npm run build:client && tsc",
    "test": "node --experimental-vm-modules node_modules/.bin/jest"
  }
}
```

This gives users clear commands:
- `npm start` — starts the server (production mode, serves built client)
- `npm run dev` — starts server with watch
- `npm run dev:client` — starts Vite dev server (for frontend development with HMR)
- `npm run build:client` — builds the React client
