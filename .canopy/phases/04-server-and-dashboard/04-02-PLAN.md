**Step 1: Create `client/src/store/useTestStore.ts`.**

This is the central state store for the entire dashboard. Follow the Zustand TypeScript pattern from the research: `create<State>()((set) => ...)`.

Define these types within the file (DO NOT import from server types — the store uses its own simplified shapes for rendering. Server types are for the SSE hook dispatch logic):

```typescript
export type ConvStatus = 'idle' | 'active' | 'completed' | 'errored';

export interface ConvRow {
  conversationId: number;
  status: ConvStatus;
  currentTurn: number;
  turnsTotal: number;
  lastLatencyMs: number;
  totalTokens: number;
  errors: number;
}

export interface LatencyPoint {
  time: number; // sequence counter (incrementing integer per SSE flush)
  p50: number;
  p95: number;
  p99: number;
}

export interface TokenPoint {
  time: number; // sequence counter
  promptTokens: number;
  completionTokens: number;
}

export interface TestConfig {
  numConversations: number;
  turnsPerConversation: number;
  concurrency: number;
}

export type TestStatus = 'idle' | 'running' | 'stopping' | 'stopped';
```

Store interface:
```typescript
interface TestStore {
  // State
  config: TestConfig;
  testStatus: TestStatus;
  conversations: Map<number, ConvRow>;
  latencyHistory: LatencyPoint[];
  tokenHistory: TokenPoint[];
  errorCounts: Record<string, number>;
  summary: unknown | null; // Will hold TestSummary shape from SSE
  throughput: { requestsPerSecond: number; tokensPerSecond: number } | null;

  // Actions
  setConfig: (cfg: Partial<TestConfig>) => void;
  setTestStatus: (s: TestStatus) => void;
  upsertConversation: (partial: Partial<ConvRow> & { conversationId: number }) => void;
  addLatencyPoint: (pt: LatencyPoint) => void;
  addTokenPoint: (pt: TokenPoint) => void;
  incrementError: (type: string) => void;
  setThroughput: (t: { requestsPerSecond: number; tokensPerSecond: number }) => void;
  setSummary: (s: unknown) => void;
  reset: () => void;
}
```

Defaults: `config: { numConversations: 5, turnsPerConversation: 5, concurrency: 5 }`.

**CRITICAL MAP MUTATION PATTERN:** In `upsertConversation`, always create a new Map to trigger re-renders:
```typescript
upsertConversation: (partial) => set((s) => {
  const convs = new Map(s.conversations);
  const existing = convs.get(partial.conversationId) ?? {
    conversationId: partial.conversationId,
    status: 'idle' as ConvStatus,
    currentTurn: 0, turnsTotal: 0, lastLatencyMs: 0, totalTokens: 0, errors: 0,
  };
  convs.set(partial.conversationId, { ...existing, ...partial });
  return { conversations: convs };
}),
```

Similarly for array state (latencyHistory, tokenHistory), always spread to a new array:
```typescript
addLatencyPoint: (pt) => set((s) => ({ latencyHistory: [...s.latencyHistory, pt] })),
```

The `reset()` action resets all state back to initial values (keeps config). Used when starting a new test.

**Step 2: Create `client/src/hooks/useSSE.ts`.**

This hook connects to `/api/events` via EventSource and dispatches events to the Zustand store.

```typescript
import { useEffect, useRef } from 'react';
import { useTestStore } from '../store/useTestStore';
```

Export function `useSSE(): void`.

Inside a `useEffect` (deps: `[]` — mount once at App level):

1. Create `EventSource('/api/events')`.
2. Store ref for cleanup.
3. On `message`: parse `JSON.parse(evt.data)` as `Array<{ type: string; [key: string]: unknown }>`. Loop through each event in the batch and dispatch:

   - `conversation:start` → `upsertConversation({ conversationId, status: 'active', turnsTotal, currentTurn: 0 })`
   - `conversation:turn:complete` → `upsertConversation({ conversationId, status: 'active', currentTurn: turnNumber, lastLatencyMs: latencyMs, totalTokens: (existing.totalTokens + promptTokens + completionTokens) })`. Also compute running percentiles from all conversations' latencies — this requires maintaining a latency accumulator. **Simpler approach:** Just track the raw latency values and compute p50/p95/p99 on each batch using a simple sorted array. Add a `LatencyPoint` with computed percentiles.

   **Actually, simpler and correct approach for latency chart:** Maintain a private array of all latencies seen so far (outside Zustand, in a module-level variable within useSSE.ts). On each `conversation:turn:complete`, push the latencyMs. Then compute p50/p95/p99 from the sorted array and call `addLatencyPoint`. Use a sequence counter for the `time` field.

   For the token chart: each `conversation:turn:complete` also carries `promptTokens` and `completionTokens`. Add a `TokenPoint` with cumulative totals tracked in a module-level accumulator.

   - `conversation:complete` → `upsertConversation({ conversationId, status: evt.status === 'completed' ? 'completed' : 'errored' })`
   - `test:lifecycle` → Map lifecycle types to TestStatus: `'starting'|'running'` → `setTestStatus('running')`, `'stopping'|'draining'` → `setTestStatus('stopping')`, `'stopped'` → `setTestStatus('stopped')`
   - `api:error` → `incrementError(evt.type as string)`
   - `metrics:summary` → `setSummary(evt.summary)`

4. On `error`: close, reconnect after 2 seconds via setTimeout.
5. Cleanup: close EventSource on unmount.

**Percentile computation helper (inline in useSSE.ts):**
```typescript
function computePercentile(sorted: number[], p: number): number {
  if (sorted.length === 0) return 0;
  const idx = Math.ceil((p / 100) * sorted.length) - 1;
  return sorted[Math.max(0, idx)];
}
```

Keep a module-level `let allLatencies: number[] = []` that resets when a `test:lifecycle` event with `type: 'starting'` arrives.

**Token accumulator:** Keep module-level `let totalPrompt = 0; let totalCompletion = 0;` — reset on `test:lifecycle` `starting`.
